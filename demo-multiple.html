Multiple instances of OpenBoxes sync'd together (v0.6.3.2)

Overview

This is a demonstration of OpenBoxes configured to replicate data across multiple server instances (powered by SymmetricDS). To see this feature at work log into each server below and follow the instructions.

The documentation to configure OpenBoxes for data replication has not been finalized yet. We'll post the instructions here as soon as we get a chance. In the meantime, please email me if you have any questions.

Notice: As of 3 November 2014 12:00 EST, the synchronization demo will be offline as we migrate to new hosting. Thank you for your patience during this transition. Please email us if you have any questions.
Status: As of 30 September 2014 17:38 EDT, synchronization services have been restored.

  • The demo servers (hosted on Google Cloud) have a tendency to be rebooted without warning and the symmetric DS service is not currently configured to start during the boot process. Therefore, if the demo is not working please create a support ticket by clicking on the in the bottom right corner (OR simply leave a comment below) and I will restart the sync services.
  • This demo is not yet configured to synchronize all database changes, but will sync basic product information, system metadata, and most transactions (including shipment, purchase orders, etc). When we get some free time, we'll update these servers soon to ensure sure that all data changes are sync'd.

Instructions

Step 1. Make a change on the parent server ...

Access parent server

1.Log onto the parent server using credentials below.
2.Choose a location.
3.Create a new product.
4.Record a stock count.
Step 2. ... then log into the child server to see the sync'd changes!

Access child server

1.Log onto the child server using credentials below.
2.Choose the same location you selected on the parent server.
3.Using the global search box at the top of the page, search for the product you just created.

Default user credentials

  • To log in as the System Administrator: admin / password
  • To log in as the Warehouse Manager: manager / password
